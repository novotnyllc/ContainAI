# fn-18-g96.8 Change container naming to repo-branch format

## Description

Replace the current hash-based container naming (`containai-<12-char-hash>`) with a human-readable `repo-branch` format. If there are duplicates (multiple workspaces with same repo-branch), add a numeric suffix.

**Current naming:**
```
containai-a1b2c3d4e5f6  # Hash of workspace path
```

**New naming:**
```
containai-myrepo-main       # First container for myrepo on main branch
containai-myrepo-main-2     # Second container if duplicate
containai-myrepo-feature-x  # Different branch
```

**What to do:**
1. Modify `_containai_container_name()` in `src/lib/container.sh`:
   - Extract repo name from workspace path (last component, or git remote name if in git repo)
   - Extract branch name from `git branch --show-current` (or "default" if not in git repo)
   - Sanitize for Docker name requirements: `[a-zA-Z0-9][a-zA-Z0-9_.-]*`
   - Format: `containai-{repo}-{branch}`
2. Add suffix handling for duplicates:
   - Check if container name already exists
   - If yes, append `-2`, `-3`, etc.
3. Update `_cai_find_container()` to handle the new naming scheme
4. Consider migration path for existing containers (document, don't auto-migrate)

**Docker name constraints:**
- Must match: `[a-zA-Z0-9][a-zA-Z0-9_.-]*`
- Max length: 64 characters (truncate if needed)
- Replace invalid chars with `-`

## Acceptance
- [ ] Container names follow `containai-{repo}-{branch}` format
- [ ] Branch name extracted from git if available
- [ ] Repo name extracted from directory or git remote
- [ ] Invalid characters replaced with `-`
- [ ] Duplicate handling with numeric suffix (-2, -3, etc.)
- [ ] Name truncated to 64 chars if too long
- [ ] Existing containers continue to work (no breaking change to lookup)
- [ ] Documentation updated with new naming scheme
