# fn-15-281.5 Update documentation for managed dockerd bundle

## Description
Update documentation to cover the managed dockerd bundle feature with versioned directories.

**Size:** S
**Files:** `docs/setup-guide.md`, `docs/troubleshooting.md`, `docs/configuration.md`

## Approach

### docs/setup-guide.md
Add section "ContainAI-managed dockerd bundle" covering:
- Bundle is installed automatically during setup (Linux/WSL2)
- Versioned structure: `/opt/containai/docker/<version>/`
- Stable symlinks: `/opt/containai/bin/`
- Updates via `cai update` (atomic via symlink swap)
- No fallback to system Docker
- System Docker is not required for installation

### docs/configuration.md
Add update configuration section covering:
- `update.check_interval` setting (hourly/daily/weekly/never, default daily)
- `CAI_UPDATE_CHECK_INTERVAL` environment variable override
- Rate-limit state stored in `~/.cache/containai/update-check`

### docs/troubleshooting.md
Add entry for "Updates available" warning:
- What the warning means (new dockerd bundle version available)
- How to apply updates: `cai update`
- **Note that dockerd updates will restart containers**
- How to disable checks: `CAI_UPDATE_CHECK_INTERVAL=never`
- Network error handling: check is non-blocking
