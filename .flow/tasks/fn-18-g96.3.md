# fn-18-g96.3 Custom user templates from config directory

## Description

Implement template directory discovery and listing. Templates are Dockerfiles that users create to customize the ContainAI image.

**Template directory resolution:**
```bash
_CAI_TEMPLATES_DIR="${XDG_CONFIG_HOME:-$HOME/.config}/containai/templates"
```

**Directory structure:**
```
$_CAI_TEMPLATES_DIR/
├── ml-gpu/
│   └── Dockerfile
├── frontend/
│   └── Dockerfile
└── nodejs/
    └── Dockerfile
```

**Changes needed:**
1. Add `_CAI_TEMPLATES_DIR` variable in `src/lib/config.sh` (alongside other config paths)
2. Add `_cai_list_templates()` function (note: `_cai_` prefix per convention)
3. Add `_cai_get_template_path()` function

**Functions (using `_cai_` prefix consistently):**
```bash
# List available template names
_cai_list_templates() {
    local templates_dir="${XDG_CONFIG_HOME:-$HOME/.config}/containai/templates"
    [[ -d "$templates_dir" ]] || return 0
    local dir
    for dir in "$templates_dir"/*/; do
        [[ -f "${dir}Dockerfile" ]] && basename "$dir"
    done
}

# Get path to template Dockerfile
_cai_get_template_path() {
    local name="$1"
    local templates_dir="${XDG_CONFIG_HOME:-$HOME/.config}/containai/templates"
    local path="$templates_dir/$name/Dockerfile"
    if [[ -f "$path" ]]; then
        printf '%s\n' "$path"
    else
        _cai_error "Template not found: $name"
        _cai_error "Templates directory: $templates_dir"
        return 1
    fi
}
```

## Acceptance
