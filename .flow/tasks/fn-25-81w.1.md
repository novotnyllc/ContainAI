# fn-25-81w.1 Extract reusable sysbox version functions in setup.sh

## Description
Extract sysbox version checking logic from `_cai_install_sysbox_linux()` into reusable functions that can be called by doctor.sh and update.sh.

Add the following functions to `src/lib/setup.sh` after the `_CAI_SYSBOX_CONTAINAI_TAG` constant (around line 466):

1. `_cai_sysbox_installed_version()` - Returns semver portion of installed sysbox (e.g., "0.6.7")
2. `_cai_sysbox_installed_pkg_version()` - Returns full dpkg version (includes +containai suffix)
3. `_cai_sysbox_is_containai_build()` - Returns 0 if installed is ContainAI build
4. `_cai_sysbox_bundled_tag()` - Returns the pinned `_CAI_SYSBOX_CONTAINAI_TAG`
5. `_cai_sysbox_bundled_version()` - Returns version from bundled release (caches result)
6. `_cai_sysbox_needs_update()` - Compares installed vs bundled, sets `_CAI_SYSBOX_UPDATE_REASON`

Version comparison logic:
- If installed is upstream and bundled is ContainAI with same/newer semver -> needs update (prefer ContainAI)
- If installed is ContainAI and bundled has newer semver -> needs update
- If installed >= bundled semver and both ContainAI -> up to date
